## What Didn't Go Well / Challenges

1. **Library API surprises at story start** — `react-native-calendar-events` (Story 5.1) and `expo-file-system` v19 `/legacy` import path (Story 5.4) were discovered mid-story. Pre-story library validation was missing.
2. **LOW technical debt accumulation** — 4 LOW-priority items carried across 4+ epics without resolution. The "LOW" label has become a deferral mechanism rather than a priority signal.
3. **Kütüphane dokümantasyonu gecikmesi** — expo-file-system v19'un breaking import path değişikliği resmi dokümantasyonda belirgin değildi.

---

## Key Insights & Lessons Learned

1. **Library validation before story start is mandatory** — Any dependency not previously used in the project must be validated (version, API surface, active maintenance) before the story implementation begins. Add this as a story template checkpoint.
2. **LOW borç = bilinçli karar, değil erteleme** — Her epic başında LOW borç listesi gözden geçirilmeli ve "şimdi kapat" veya "erteliyorum çünkü X" kararı aktif olarak alınmalı.
3. **Expo version pinning awareness** — Major expo-\* package versions may have breaking API changes not prominently documented. Check changelog on any expo package update.
4. **Premium aşamaya hazırlık = sıfır hata payı** — GOZE'nin vurguladığı gibi: Epic 6'da kullanıcı para ödüyor. Unprofessional görünmenin maddi karşılıkları var. Hazırlık kritik, sürpriz kabul edilemez.
